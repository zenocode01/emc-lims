## 🔴 严重问题（生产环境会失败）

### 1. SysRole 实体字段与 init.sql 列名不匹配

**严重程度**: 🔴 高  
**影响范围**: 角色管理的所有 CRUD 操作

| 实体字段（Java） | MyBatis-Plus 映射列名 | init.sql 实际列名 | 是否匹配 |
|---|---|---|---|
| `roleName` | `role_name` ❌ | `name` ✅ | ❌ 不匹配 |
| `roleCode` | `role_code` ❌ | `code` ✅ | ❌ 不匹配 |
| `roleDesc` | `role_desc` ❌ | `type` ✅ | ❌ 不匹配 |

**问题描述**：
- `SysRole.java` 实体字段未使用 `@TableField` 注解指定数据库列名
- MyBatis-Plus 自动将 camelCase 字段名转为 snake_case 列名
- `init.sql` 中使用的列名与实体期望的列名不一致

**影响**：INSERT/UPDATE/DELETE 操作会在生产数据库报错 `column "role_name" does not exist`

**涉及文件**：
- 实体：`backend/src/main/java/com/emclims/module/sys/entity/SysRole.java`
- 数据库：`docker/init.sql` 第 59-77 行

**当前代码**：
```java
// SysRole.java 当前
private String roleName;      // MyBatis-Plus 映射到 role_name
private String roleCode;      // MyBatis-Plus 映射到 role_code
private String roleDesc;      // MyBatis-Plus 映射到 role_desc
```

```sql
-- docker/init.sql 实际
CREATE TABLE sys_role (
    id          BIGINT PRIMARY KEY,
    name        VARCHAR(50) NOT NULL,      -- 不是 role_name
    code        VARCHAR(50) NOT NULL UNIQUE, -- 不是 role_code
    type        SMALLINT DEFAULT 2,         -- 不是 role_desc
    ...
);
```

**修复方案**（二选一）：

- **方案 A**：在实体字段加 `@TableField` 注解
  ```java
  @TableField("name")
  private String roleName;
  
  @TableField("code")
  private String roleCode;
  
  @TableField("type")
  private String roleDesc;
  ```

- **方案 B**：修改 `docker/init.sql` 列名为 `role_name`, `role_code`, `role_desc`

### 5. SysMenu `isHidden` 语义反转

**严重程度**: 🟡 中  
**影响范围**：前端菜单显示/隐藏逻辑

| 实体字段 | 含义 | 数据库列 | 含义 | 是否反转 |
|---|---|---|---|---|
| `isHidden` | 1=隐藏，0=显示 | `visible` | 1=显示，0=隐藏 | ✅ 反转 |

**问题描述**：
- 实体字段命名为 `isHidden`（isHidden=1 表示隐藏）
- 数据库列名为 `visible`（visible=1 表示显示）
- 两者逻辑相反，直接映射会导致显示/隐藏状态错误

**影响**：前端显示/隐藏菜单的逻辑可能与预期相反

**涉及文件**：
- 实体：`backend/src/main/java/com/emclims/module/sys/entity/SysMenu.java`
- 数据库：`docker/init.sql` 第 90-112 行

**修复方案**：
```java
// 方案 A：在 Service 层做逻辑转换
@TableField("visible")
private Integer isHidden;

// 在 Service 中：
// entity.setHidden(menu.getVisible() != null ? (menu.getVisible() == 0 ? 1 : 0) : null);

// 方案 B：重命名字段（更推荐）
@TableField("visible")
private Integer visible;  // 与数据库列名一致，避免混淆
```

### 6. 缺少 `SecurityFilterChain` 配置

**严重程度**: 🟡 中  
**影响范围**：所有 API 请求的认证/授权流程

**问题描述**：
- 项目中使用了 `JwtAuthenticationFilter`（自定义 JWT 过滤器）
- 但没有 `SecurityConfig` 配置 `SecurityFilterChain`
- Spring Security 使用默认配置（可能拦截所有请求）
- 过滤器注册顺序不确定

**影响**：
- 可能拦截 `/auth/login` 登录接口
- 过滤器执行顺序不可控
- `JwtAuthenticationFilter` 的 `@Component` 注解依赖 Spring 自动注册，顺序不确定

**涉及文件**：
- 过滤器：`backend/src/main/java/com/emclims/common/security/JwtAuthenticationFilter.java`
- 缺失配置：`SecurityConfig.java`（不存在）

**修复方案**：
```java
@Configuration
public class SecurityConfig {
    @Autowired
    private JwtAuthenticationFilter jwtFilter;
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()
                .requestMatchers("/api/doc.html", "/api/webjars/**").permitAll()
                .anyRequest().authenticated())
            .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)
            .build();
    }
}
```

**问题描述**：

- 测试环境与生产环境配置不一致

**配置位置**：`backend/src/main/resources/application.yml`

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    password:     # 空
```

**修复方案**：
- 开发环境：保持为空（如果 Redis 无密码）
- 生产环境：使用环境变量或配置文件外部化
  ```yaml
  spring:
    redis:
      password: ${REDIS_PASSWORD}
  ```
